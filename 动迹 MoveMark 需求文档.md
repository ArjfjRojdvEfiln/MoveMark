### 动迹 MoveMark 需求文档 v1.0

#### 一、项目概述

面向跑步、骑行、越野等户外运动爱好者的赛事聚合平台，提供赛事搜索、收藏订阅、AI备赛问答三大核心功能。数据来源于公开赛事平台爬取的真实数据。

------

#### 二、用户角色

只有一个角色：**注册用户**（游客可浏览，登录后可收藏和使用AI）

------

#### 三、核心功能模块

**模块1：赛事广场**

- 赛事列表，支持分页
- 多条件筛选：运动类型（马拉松/骑行/越野）、省份、月份、报名状态
- ES全文搜索：按赛事名称、城市、关键词搜索
- 赛事详情页：时间、地点、报名时间、赛道距离、官网链接

**模块2：用户中心**

- 注册/登录，JWT鉴权
- 收藏赛事
- 头像上传（OSS）
- 我的收藏列表

**模块3：赛事提醒**

- 收藏赛事后，RabbitMQ异步发送站内提醒
- 赛事报名即将截止时（提前3天），自动推送提醒消息
- 用户消息中心页展示通知列表

**模块4：AI备赛助手**

- Dify驱动，RAG知识库包含：爬取的赛事数据 + 训练知识文档
- 用户可对话式提问："我是初跑者，想参加半马怎么准备？"
- 上下文多轮对话，有记忆

------

#### 四、技术点对应说明（答辩要讲清楚）

| 技术        | 具体用途                   | 答辩能说的点                           |
| ----------- | -------------------------- | -------------------------------------- |
| FastAPI     | 后端接口                   | 异步框架，性能好，自动生成API文档      |
| ORM + MySQL | 数据持久化                 | 关系型数据，结构化存储赛事、用户、收藏 |
| Redis       | 赛事列表缓存、热门赛事排行 | 减少DB查询，缓存击穿用布隆过滤器防护   |
| ES          | 赛事全文搜索、多条件筛选   | 倒排索引，比MySQL LIKE快，支持geo查询  |
| OSS         | 用户头像、赛事图片存储     | 非结构化文件不入库，分布式存储         |
| RabbitMQ    | 收藏后异步通知、截止提醒   | 解耦、削峰，通知不阻塞主流程           |
| Dify        | AI对话+RAG知识库           | 深度集成，不是简单套壳                 |
| 布隆过滤器  | 爬虫去重 + 缓存穿透防护    | 空间效率高，两个场景都用到             |
| JWT         | 登录鉴权，区分游客/用户    | 无状态，适合前后端分离                 |
| Vue3 + TS   | 前端                       | 组合式API，类型安全                    |

------

#### 五、数据来源

- **跑步天地** / **美丽跑** / **中国田径协会官网** 爬取赛事数据
- 字段：赛事名、类型、省份、城市、比赛日期、报名截止日、距离组别、官网URL、赛事图片
- 预计入库 **200-500条**真实赛事数据，足够展示
